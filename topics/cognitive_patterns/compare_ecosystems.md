# Compare Ecosystems

## Purpose

Use this pattern to compare regions, industries, organizations, or platforms without reducing the comparison to headline metrics.

## Use when

- One ecosystem consistently outperforms another.
- You want to transfer lessons between domains.
- A policy proposal assumes a foreign model can be copied.
- Different systems produce similar outputs through different mechanisms.

## Comparison dimensions

1. Actors
2. Density and proximity
3. Capital
4. Infrastructure
5. Talent and skills
6. Standards and interfaces
7. Demand
8. Governance
9. Information flows
10. Learning speed
11. Entry and exit dynamics
12. Cultural status and incentives

## Procedure

1. Define the unit of comparison.
2. Clarify the outcome each ecosystem is optimized for.
3. Map key actors and their dependencies.
4. Compare resource and information flows.
5. Compare feedback-loop speed.
6. Compare modularity and standards.
7. Identify where learning occurs.
8. Compare state capacity and governance.
9. Distinguish transferable mechanisms from context-specific features.
10. State where the analogy breaks.

## Comparison table

| Dimension | Ecosystem A | Ecosystem B | Implication |
|---|---|---|---|
| Supplier density | High | Low | Faster iteration in A |
| Anchor demand | Strong | Weak | Easier scaling in A |
| Grid access | Fast | Slow | Lower deployment friction in A |

## Failure modes

- Comparing countries using only aggregate statistics.
- Assuming visible institutions are the true cause.
- Copying practices without complementary capabilities.
- Ignoring historical path dependence.
- Selecting only similarities and overlooking structural differences.
- Treating culture as either irrelevant or all-explanatory.

## Example

### Shenzhen versus a dispersed manufacturing base

The useful comparison is not simply labour cost. It includes supplier density, lead times, iteration speed, tool availability, anchor customers, and the status of technical work.

## Evaluation

A good ecosystem comparison should:

- identify mechanisms, not just differences;
- include at least one learning-speed measure;
- distinguish portable from non-portable lessons;
- explicitly state where the analogy fails;
- produce concrete implications for system design.
