# Analyze Capability Accumulation

## Purpose

Use this pattern to explain why some people, firms, regions, or institutions become progressively better at doing difficult things.

The focus is not only on current resources or outputs, but on the mechanisms through which capability compounds.

## Use when

- A region or organization appears unusually effective.
- You want to explain durable competitive advantage.
- The problem involves manufacturing, engineering, operations, innovation, or institutional competence.
- Static explanations such as capital, labour cost, or talent seem incomplete.

## Core questions

1. Where is practical knowledge created?
2. How quickly does feedback return to the people doing the work?
3. Which activities generate learning-by-doing?
4. How dense are the interactions between specialists?
5. What tacit knowledge would be lost if production stopped?
6. Which capabilities reinforce one another?
7. Which bottlenecks slow accumulation?
8. What causes the system to retain or lose capability?

## Procedure

1. Define the capability precisely.
2. Identify the actors who produce, use, and transmit it.
3. Map the learning loop from action to feedback to improvement.
4. Identify complementary capabilities and infrastructure.
5. Separate explicit knowledge from tacit knowledge.
6. Look for increasing returns, path dependence, and lock-in.
7. Identify what accelerates or interrupts compounding.
8. Compare the system with a weaker or declining counterpart.
9. State the main capability-building mechanism.
10. Propose interventions that strengthen the loop rather than only subsidize outputs.

## Typical outputs

- Capability map
- Learning-loop diagram
- Bottleneck list
- Compounding mechanism
- Policy or organizational implications

## Failure modes

- Confusing current scale with capability.
- Treating capital investment as sufficient.
- Ignoring tacit knowledge and supplier networks.
- Assuming capability transfers instantly between regions.
- Explaining success only through culture.
- Ignoring the possibility that capability can decay.

## Example

### Question

Why did Shenzhen become dominant in electronics manufacturing?

### Analysis

- Dense supplier network shortened iteration cycles.
- Repeated production created tacit process knowledge.
- Engineers and suppliers exchanged feedback rapidly.
- Specialized firms improved independently while reinforcing the cluster.
- Scale increased both demand and the number of viable specialists.

### Conclusion

Shenzhen's advantage is not merely factory count. It is a capability-accumulation system with unusually fast feedback and knowledge diffusion.

## Evaluation

A good analysis should:

- explain how capability grows over time;
- identify at least one reinforcing feedback loop;
- distinguish capability from output;
- explain why the capability is difficult to copy;
- identify conditions under which the capability could decay.
