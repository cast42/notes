# Thinking in Systems (Donella Meadows)

## TL;DR
A foundational primer that makes systems thinking practical: stocks, flows, feedback loops, delays, resilience, and leverage points. Meadows’ core argument is that many persistent societal problems are system-structure problems, so durable solutions require changing the system, not only reacting to events.

## Key takeaways
- **Structure drives behavior**: outcomes emerge from feedback + delays, not isolated events.
- **Stocks and flows first**: if you can’t map accumulations and rates, you’ll misdiagnose problems.
- **Delays create policy surprises**: good intentions can fail when response times are mismatched.
- **Leverage points exist, but are counterintuitive**: changing goals/information flows often beats parameter tweaks.
- **Humility is part of the method**: system boundaries, assumptions, and models are always partial.
