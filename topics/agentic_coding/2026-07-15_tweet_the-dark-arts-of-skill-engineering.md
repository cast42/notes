# The Dark Arts of Skill Engineering

*Paul Bakaus — @pbakaus*

## TL;DR

- Paul Bakaus argues that most skills are not really interoperable prompt snippets but harness extensions that must reliably change model behavior across runtimes, projects, and user styles.
- His core move is to shift skill engineering from prose-only prompting toward machinery: routing, memory, scripts, hooks, browser wiring, adversarial review, and guardrails that force outcomes instead of merely requesting them.
- The essay is a practical playbook for building robust agent skills, with nine techniques aimed at making non-deterministic models behave more like dependable software.

## Highlights

- "Most skills are a prompt and a prayer."
- A skill should be treated as a plugin or harness extension, not just a saved prompt.
- "Prompting summons the median; the harness conjures the rest."
- Two blind reviewers beat one confident guess, which is why adversarial or independent sub-agents can improve review quality.
- The broader thesis is that reliable skills move work from prose into enforceable machinery.
