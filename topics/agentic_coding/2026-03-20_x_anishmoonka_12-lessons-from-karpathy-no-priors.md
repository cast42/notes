## TL;DR
Anish Moonka shared a long thread distilling Karpathy’s No Priors podcast into practical lessons about agentic coding, research automation, and where AI leverage is moving.

Core pattern: humans are shifting from writing lines of code to orchestrating parallel agents, while evaluation loops and task framing become the bottlenecks.

## Key takeaways
- Claimed "December 2025 flip": rapid shift from direct coding to agent orchestration.
- Unit of work moved from lines/functions to feature-level delegation.
- Throughput bottleneck is increasingly human task-definition, not model capability.
- Agent personality/memory can materially affect usage and trust.
- AutoResearch-style loops can discover non-obvious optimization interactions at scale.
- Open source frontier gap is framed as narrowing.
- Digital (bits) disruption likely lands before physical (atoms/robotics).
- Jevons-style effect in software: lower production cost can increase total software demand.

## Lessons captured in the thread (condensed)
1. A real workflow flip happened in late 2025 for some top builders.
2. Engineers increasingly manage multiple agents in parallel by feature.
3. Idle token/subscription capacity = lost throughput.
4. Productive agent UX includes personality + memory, not only raw capability.
5. AutoResearch can run huge experiment volumes and surface missed optimizations.
6. Research org behavior can be encoded as instruction artifacts and iterated.
7. Models can be superhuman in narrow domains while weak in others.
8. Open-source model gap versus frontier is shrinking.
9. Digital systems will transform faster than physical systems.
10. Cheaper software creation can increase software demand (Jevons framing).
11. Distributed/untrusted compute swarms could compete with centralized labs if verification works.
12. Teaching may shift toward creating agent-readable skills/curricula.
