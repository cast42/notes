# Control the ideas, not the code

*antirez*

## TL;DR

- Antirez argues that AI has shifted the programmer’s bottleneck away from line-by-line code reading and toward design, testing, and control of the software’s underlying ideas.
- His point is not blind vibe coding, but that once models can generate large volumes of competent local code, humans create more value by steering architecture, checking conceptual fit, and doing strong QA.
- The practical implication is that design artifacts and mental models may become more useful than exhaustive code review, especially in fast-moving technical domains.

## Highlights

- "Focus on controlling the ideas, instead."
- The post argues that scanning generated code function by function is increasingly a bad tradeoff when design leverage is higher elsewhere.
- Antirez says LLMs are often good at locally optimal code but weaker on big ideas, so human attention should stay anchored at the systems-design level.
- He proposes DESIGN.md-style human-readable descriptions of data structures and implementation tricks as a more useful long-term artifact.
- For younger programmers, he still sees value in learning by building interpreters, databases, and data structures rather than just reviewing incidental AI-generated application code.
