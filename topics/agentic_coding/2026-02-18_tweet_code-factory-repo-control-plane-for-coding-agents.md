# Code Factory — repo control-plane for coding agents

*Ryan Carson — @ryancarson*

## TL;DR

- Link-post to an X Article: "Code Factory" — how to wire a repo so an agent can write code, pass risk-aware gates, and get machine-verifiable evidence.
- Core loop: agent writes → policy gate/preflight → CI/tests/browser evidence → review agent validates PR.
- Key idea: one machine-readable contract (risk tiers, required checks, docs drift rules, evidence requirements) to prevent silent workflow drift.

## Highlights

- Use a repo "control plane" (policy + checks) so agents can operate safely and repeatably.
- Gate preflight before expensive CI; keep evidence requirements explicit.
- Review agent options mentioned: @greptile, @coderabbitai, CodeQL/policy logic, custom LLM review.
- Credits inspiration to @_lopopolo.
