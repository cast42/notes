# markdownfs, an open source in-memory filesystem for agents

*Subramanya N — @subramanya*

## TL;DR

- A short X reply points to markdownfs as an open-source version of an agent-oriented in-memory filesystem, with most of the substance living in the linked GitHub repo.
- markdownfs is a Rust-built markdown-native virtual filesystem and database with CLI, HTTP, and MCP interfaces, plus search, permissions, persistence, and Git-like versioning.
- The interesting idea is not just speed or storage, but making agent memory and workspace state inspectable, durable, revertible, and shareable through plain markdown files.

## Highlights

- markdownfs treats Markdown as the primary storage substrate for agent workspaces rather than as a final export format.
- Its MCP server makes the system directly relevant to AI agents that need read, write, search, commit, and rollback over shared artifacts.
- This is a strong example of the broader move toward transparent, file-native memory layers for agents instead of opaque vector-only or app-specific stores.
